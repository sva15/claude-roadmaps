## 1. First Principles — Start Simple

### What Problem Does This Solve?

Terraform has 100+ built-in functions. You don't need to memorize them all. But there are ~15 that appear in **virtually every production project**. Not knowing them means you'll be constantly looking up docs mid-project.

These functions solve real problems:
- How do you calculate subnet CIDRs from a VPC CIDR? → `cidrsubnet()`
- How do you merge tag maps from different sources? → `merge()`
- How do you build an IAM policy in HCL? → `jsonencode()`
- How do you flatten a nested list for `for_each`? → `flatten()` + `toset()`
- How do you safely access an optional value? → `try()` / `can()` / `coalesce()`

## 2. Core Concept — Deep Dive

### Category 1: Network Functions (CIDR Math)

These are the #1 "I didn't know Terraform had this" functions. They eliminate manual CIDR calculations.

#### `cidrsubnet(prefix, newbits, netnum)` — Calculate Subnet CIDRs

```hcl
# VPC CIDR: 10.0.0.0/16 (65,536 IPs)
# I want /24 subnets (256 IPs each)
# newbits = 24 - 16 = 8

cidrsubnet("10.0.0.0/16", 8, 0)    # → "10.0.0.0/24"   (1st subnet)
cidrsubnet("10.0.0.0/16", 8, 1)    # → "10.0.1.0/24"   (2nd subnet)
cidrsubnet("10.0.0.0/16", 8, 10)   # → "10.0.10.0/24"  (11th subnet)
cidrsubnet("10.0.0.0/16", 8, 255)  # → "10.0.255.0/24" (last /24)

# Real usage: create subnets dynamically
resource "aws_subnet" "public" {
  for_each   = { for i, az in local.azs : az => i }
  cidr_block = cidrsubnet(var.vpc_cidr, 8, each.value)      # Public: 10.0.0.0/24, 10.0.1.0/24
}

resource "aws_subnet" "private" {
  for_each   = { for i, az in local.azs : az => i }
  cidr_block = cidrsubnet(var.vpc_cidr, 8, each.value + 10)  # Private: 10.0.10.0/24, 10.0.11.0/24
}
```

**How to read it:** "Take this network, add 8 bits to the prefix, give me subnet number N."

#### `cidrsubnets(prefix, newbits...)` — Multiple Subnets at Once

```hcl
# Calculate 4 subnets with different sizes
cidrsubnets("10.0.0.0/16", 8, 8, 4, 4)
# → ["10.0.0.0/24", "10.0.1.0/24", "10.0.0.0/20", "10.0.16.0/20"]
#    Small (256 IPs)  Small         Large (4096)    Large

# Real usage: different-sized subnets for different tiers
locals {
  subnet_cidrs = cidrsubnets(var.vpc_cidr, 8, 8, 8, 8, 4, 4)
  # [0-3]: small public subnets (/24)
  # [4-5]: large private subnets (/20)
}
```

#### `cidrhost(prefix, hostnum)` — Get a Specific IP

```hcl
cidrhost("10.0.0.0/24", 5)     # → "10.0.0.5"
cidrhost("10.0.0.0/24", -1)    # → "10.0.0.254" (last usable)

# Used in validation:
validation {
  condition     = can(cidrhost(var.vpc_cidr, 0))
  error_message = "Must be a valid CIDR block."
}
```

### Category 2: Collection Functions

#### `merge(map1, map2, ...)` — Merge Maps (Last Wins)

```hcl
# Merge tag maps — used in EVERY module
locals {
  default_tags = {
    Environment = var.environment
    ManagedBy   = "terraform"
  }
  
  resource_tags = {
    Name = "${local.name_prefix}-vpc"
    Tier = "networking"
  }
}

resource "aws_vpc" "main" {
  tags = merge(local.default_tags, local.resource_tags)
  # Result: { Environment = "dev", ManagedBy = "terraform", Name = "...-vpc", Tier = "networking" }
}

# Later keys override earlier:
merge({ a = 1, b = 2 }, { b = 3, c = 4 })
# → { a = 1, b = 3, c = 4 }  (b=3 wins over b=2)
```

#### `flatten(list_of_lists)` — Flatten Nested Structures

```hcl
# THE for_each flattening pattern (used constantly)
variable "subnet_config" {
  default = {
    public  = ["10.0.0.0/24", "10.0.1.0/24"]
    private = ["10.0.10.0/24", "10.0.11.0/24"]
  }
}

locals {
  all_subnets = flatten([
    for tier, cidrs in var.subnet_config : [
      for i, cidr in cidrs : {
        key  = "${tier}-${i}"
        tier = tier
        cidr = cidr
      }
    ]
  ])
  # Result: flat list of 4 objects
  # Convert to map for for_each:
  subnet_map = { for s in local.all_subnets : s.key => s }
}

resource "aws_subnet" "all" {
  for_each   = local.subnet_map
  cidr_block = each.value.cidr
  tags       = { Tier = each.value.tier }
}
```

#### `toset(list)` — Convert List to Set for for_each

```hcl
# for_each needs a SET or MAP, not a list
variable "allowed_ports" {
  type    = list(number)
  default = [80, 443, 8080]
}

resource "aws_security_group_rule" "ingress" {
  for_each    = toset([for p in var.allowed_ports : tostring(p)])
  type        = "ingress"
  from_port   = tonumber(each.value)
  to_port     = tonumber(each.value)
  protocol    = "tcp"
  cidr_blocks = ["0.0.0.0/0"]
  security_group_id = aws_security_group.main.id
}
```

#### `lookup(map, key, default)` — Safe Map Access

```hcl
variable "instance_types" {
  default = {
    dev  = "t3.micro"
    prod = "t3.large"
  }
}

# Safe: returns "t3.small" if key doesn't exist
instance_type = lookup(var.instance_types, var.environment, "t3.small")

# Without lookup: var.instance_types[var.environment]
# → ERROR if var.environment = "staging" (key doesn't exist!)
```

#### `length(collection)` — Count Elements

```hcl
# Used in validations, conditionals, and count
length(["a", "b", "c"])      # → 3
length({ a = 1, b = 2 })     # → 2
length("hello")               # → 5

# Common validation pattern
validation {
  condition     = length(var.vpc_cidr) > 0
  error_message = "VPC CIDR cannot be empty."
}

# Conditional resource creation
count = length(var.private_subnets) > 0 ? 1 : 0
```

### Category 3: Encoding Functions

#### `jsonencode(value)` — HCL to JSON

```hcl
# THE IAM policy pattern — used in every IAM-related config
resource "aws_iam_role_policy" "app" {
  role   = aws_iam_role.app.name
  policy = jsonencode({
    Version = "2012-10-17"
    Statement = [
      {
        Effect   = "Allow"
        Action   = ["s3:GetObject", "s3:PutObject"]
        Resource = "${aws_s3_bucket.app.arn}/*"
      },
      {
        Effect   = "Allow"
        Action   = ["secretsmanager:GetSecretValue"]
        Resource = aws_secretsmanager_secret.db.arn
      }
    ]
  })
}
```

**Why not heredoc JSON?** `jsonencode` lets you interpolate Terraform expressions (ARNs, IDs) directly. With heredoc JSON, you need string interpolation which is error-prone.

#### `jsondecode(string)` — JSON to HCL

```hcl
# Parse JSON from a data source or file
locals {
  config = jsondecode(file("${path.module}/config.json"))
}

# Access values
resource "aws_instance" "web" {
  instance_type = local.config.instance_type
  tags          = local.config.tags
}
```

### Category 4: String Functions

#### `format(format_string, args...)` — String Formatting

```hcl
format("arn:aws:s3:::%s/*", var.bucket_name)
# → "arn:aws:s3:::my-bucket/*"

format("%s-%s-%s", var.project, var.environment, "vpc")
# → "myapp-prod-vpc"

# Padding/alignment
format("%-20s %s", "Name:", var.name)
```

#### `templatefile(path, vars)` — Render Templates

```hcl
# user_data.sh.tpl:
# #!/bin/bash
# echo "DB_HOST=${db_host}" >> /etc/environment
# echo "DB_PORT=${db_port}" >> /etc/environment

resource "aws_instance" "web" {
  user_data = templatefile("${path.module}/user_data.sh.tpl", {
    db_host = aws_db_instance.main.endpoint
    db_port = aws_db_instance.main.port
  })
}
```

**Use templatefile() over heredoc** when templates are more than 5 lines — it keeps .tf files clean and templates testable.

### Category 5: Type Handling & Safety Functions

#### `try(expression, fallback)` — Safe Attribute Access

```hcl
# Without try: crash if attribute doesn't exist
# With try: return fallback value

name = try(var.config.name, "default-name")
port = try(var.service.ports.http, 80)

# Multi-level fallback
region = try(var.override_region, var.default_region, "us-east-1")

# Real usage: optional module inputs
cidr = try(var.custom_cidr, cidrsubnet(var.vpc_cidr, 8, 0))
```

#### `can(expression)` — Boolean Test

```hcl
# Returns true/false instead of the value — used in validations
validation {
  condition     = can(cidrhost(var.vpc_cidr, 0))
  error_message = "Must be a valid CIDR block."
}

validation {
  condition     = can(regex("^t[23]\\.", var.instance_type))
  error_message = "Only t2/t3 instance families allowed."
}
```

#### `coalesce(val1, val2, ...)` — First Non-Null

```hcl
# Returns the first argument that is not null or empty string
region = coalesce(var.override_region, var.default_region, "us-east-1")
# If override_region is null → checks default_region → falls back to "us-east-1"

# Difference from try():
# try()     → catches ERRORS (attribute doesn't exist)
# coalesce() → finds first non-null VALUE (all attributes exist, some are null)
```

## 3. Function Cheat Sheet

```hcl
# ─── NETWORK ───
cidrsubnet("10.0.0.0/16", 8, 0)       # → "10.0.0.0/24"
cidrsubnets("10.0.0.0/16", 8, 8, 4)   # → ["10.0.0.0/24", "10.0.1.0/24", "10.0.0.0/20"]
cidrhost("10.0.0.0/24", 5)            # → "10.0.0.5"

# ─── COLLECTIONS ───
merge(map1, map2)                      # Merge maps (last wins)
flatten([[1,2],[3,4]])                 # → [1,2,3,4]
toset(["a","b"])                       # List → Set (for for_each)
lookup(map, key, default)              # Safe map access
length(collection)                     # Count elements
keys(map)                              # → list of keys
values(map)                            # → list of values
contains(list, value)                  # → true/false
concat(list1, list2)                   # Merge lists
distinct(list)                         # Remove duplicates
sort(list)                             # Sort strings
slice(list, start, end)               # Sublist

# ─── ENCODING ───
jsonencode(value)                      # HCL → JSON string
jsondecode(string)                     # JSON string → HCL
yamlencode(value)                      # HCL → YAML
file(path)                             # Read file contents
templatefile(path, vars)               # Render template

# ─── STRING ───
format(fmt, args...)                   # Printf-style formatting
join(sep, list)                        # → "a-b-c"
split(sep, string)                     # → ["a","b","c"]
replace(string, search, replace)       # String replacement
regex(pattern, string)                 # Regex match
upper(string) / lower(string)          # Case conversion
trimspace(string)                      # Strip whitespace
startswith(string, prefix)             # → true/false
endswith(string, suffix)               # → true/false

# ─── TYPE/SAFETY ───
try(expr, fallback)                    # Safe access (catches errors)
can(expr)                              # → true/false (for validations)
coalesce(val1, val2, ...)              # First non-null value
tostring(num) / tonumber(str)          # Type conversion
tobool(string)                         # String → bool
```

## 10. Interview Preparation

### Q1: *"How do you calculate subnet CIDRs in Terraform?"* (Mid)

**Strong Answer:**
"`cidrsubnet(prefix, newbits, netnum)`. For a `/16` VPC wanting `/24` subnets, it's `cidrsubnet(var.vpc_cidr, 8, 0)` for the first subnet, incrementing `netnum` for each additional subnet. The `8` is `24 - 16` — the additional prefix bits. I offset private subnets by 10 (`netnum + 10`) to create logical separation. For variable-sized subnets, `cidrsubnets` takes multiple newbit arguments and returns all CIDRs at once."

### Q2: *"How do you build IAM policies in Terraform?"* (Mid)

**Strong Answer:**
"`jsonencode()` to convert HCL objects to JSON inline. This lets me reference Terraform values directly — ARNs, bucket names, account IDs — without string interpolation in heredoc JSON. For complex policies, I use `templatefile()` with a `.json.tpl` template. I always use `jsonencode` over raw JSON strings to get HCL syntax validation."

### Q3: *"Explain the flatten + for_each pattern."* (Senior)

**Strong Answer:**
"When you have nested data — like a map of lists — `for_each` needs a flat map. The pattern: use nested `for` expressions inside `flatten()` to create a flat list of objects, each with a unique `key` field. Then convert to a map with `{ for item in list : item.key => item }`. This gives `for_each` unique string keys that map to stable state addresses."

### Q4: *"Difference between `try`, `can`, and `coalesce`?"* (Senior)

**Strong Answer:**
"`can()` returns a boolean — it tests whether an expression would succeed. Used in variable validations: `can(cidrhost(var.cidr, 0))`. `try()` returns the expression's value if it succeeds, or a fallback value if it errors. Used for optional attributes: `try(var.config.name, 'default')`. `coalesce()` returns the first non-null argument — it doesn't catch errors, just picks the first value that exists. Used for optional overrides: `coalesce(var.override, var.default, 'fallback')`."
