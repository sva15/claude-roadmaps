## 1. First Principles — Start Simple

### What Problem Does This Solve?

At scale, you have hundreds of resources across multiple teams and environments. Without tags:
- **Cost allocation** — Which team's EC2 instances cost $10K last month? Unknown.
- **Compliance** — Are all production databases encrypted? Can't filter without tags.
- **Automation** — Auto-delete dev resources at night? Can't target without tags.
- **Incident response** — This IP belongs to which application? No way to know.

Tags are **metadata** on AWS resources. Terraform ensures every resource gets the right tags, consistently.

## 2. Core Concept — Deep Dive

### The Tagging Strategy

| Tag | Purpose | Example | Required? |
|-----|---------|---------|-----------|
| `Environment` | Cost filtering, access control | `dev`, `staging`, `prod` | ✅ Always |
| `Project` | Cost allocation per project | `payment-service` | ✅ Always |
| `ManagedBy` | Distinguish IaC vs manual | `terraform` | ✅ Always |
| `Team` / `Owner` | Contact for incidents | `platform-team` | ✅ Always |
| `CostCenter` | Finance/billing mapping | `CC-1234` | ✅ For production |
| `Tier` | Resource classification | `public`, `private` | Optional |
| `Backup` | Backup policy selection | `daily`, `weekly`, `none` | Optional |
| `Compliance` | Regulatory marking | `hipaa`, `pci`, `none` | If applicable |

### Implementation with default_tags (Best Practice)

```hcl
# providers.tf — tags applied to ALL resources automatically
provider "aws" {
  region = var.aws_region

  default_tags {
    tags = {
      Environment = var.environment
      Project     = var.project_name
      ManagedBy   = "terraform"
      Team        = var.team_name
      CostCenter  = var.cost_center
    }
  }
}
```

**With `default_tags`, you don't need to add these tags to every resource.** They're applied automatically by the provider.

**Resource-specific tags are merged on top:**
```hcl
resource "aws_vpc" "main" {
  cidr_block = var.vpc_cidr

  tags = {
    Name = "${local.name_prefix}-vpc"   # Resource-specific
    Tier = "networking"                  # Resource-specific
  }
  # Final tags: Environment, Project, ManagedBy, Team, CostCenter + Name + Tier
}
```

### Module-Level Tagging Pattern

```hcl
# Module accepts additional tags
variable "additional_tags" {
  type    = map(string)
  default = {}
}

locals {
  common_tags = merge({
    Module      = "vpc"
    Environment = var.environment
  }, var.additional_tags)
}

resource "aws_vpc" "main" {
  tags = merge(local.common_tags, {
    Name = "${local.name_prefix}-vpc"
  })
}
```

### Enforcing Tags with AWS

```hcl
# AWS Organizations Tag Policy (enforces tag keys/values across accounts)
# Configured separately — works alongside Terraform tagging

# AWS Config Rule (alerts on untagged resources)
resource "aws_config_config_rule" "required_tags" {
  name = "required-tags"

  source {
    owner             = "AWS"
    source_identifier = "REQUIRED_TAGS"
  }

  input_parameters = jsonencode({
    tag1Key   = "Environment"
    tag1Value = "dev,staging,prod"
    tag2Key   = "Project"
    tag3Key   = "ManagedBy"
  })
}
```

## 3. Cost Allocation

### How Tags Enable Cost Tracking

```
AWS Cost Explorer:
  Filter: Tag "Project" = "payment-service"
  Group by: Tag "Environment"
  
  Result:
    dev:     $1,200/month
    staging: $2,400/month
    prod:    $18,000/month
```

### Cost Allocation Tags Setup

```bash
# In AWS Billing Console → Cost Allocation Tags
# 1. Activate the tags you want to use for cost reporting
# 2. Takes ~24 hours for tags to appear in billing reports

# Tags must be:
# - Consistent across all resources (Terraform ensures this)
# - Activated in the billing console
# - Applied BEFORE the billing period (retroactive tagging doesn't backfill costs)
```

## 10. Interview Preparation

### Q1: *"How do you implement a tagging strategy in Terraform?"* (Mid)

**Strong Answer:**
"Two layers: `default_tags` in the provider block for organization-wide tags like Environment, Project, ManagedBy, and Team. These apply to every resource automatically. Then resource-specific tags for things like Name and Tier. For modules, I merge `additional_tags` variable with module-specific tags. On the AWS side, I use AWS Config rules to detect untagged resources and tag policies in AWS Organizations to enforce allowed values."

### Q2: *"A resource in production has no Cost Center tag. How did it happen and how do you prevent it?"* (Senior)

**Strong Answer:**
"If Terraform manages it, someone either didn't include CostCenter in `default_tags` or removed it during a change. Prevention: `default_tags` in the provider ensures it's always applied. Backup: AWS Config rule alerts on missing tags. For non-Terraform resources: AWS Organizations Service Control Policy can deny `CreateResource` actions that don't include required tags."

## 13. Cheat Sheet

```hcl
# Provider-level (all resources get these)
provider "aws" {
  default_tags {
    tags = {
      Environment = var.environment
      Project     = var.project_name
      ManagedBy   = "terraform"
      Team        = var.team_name
    }
  }
}

# Resource-level (merged with default_tags)
resource "r" "x" {
  tags = { Name = "my-resource" }
}

locals {
  tags = merge(var.common_tags, { Module = "vpc" })
}
```
