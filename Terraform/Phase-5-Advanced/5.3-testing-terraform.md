## 1. First Principles — Start Simple

### What Problem Does This Solve?

Infrastructure code can break production just like application code. But unlike applications:
- Testing infrastructure **costs real money** (creating VPCs, EC2 instances)
- Tests take **minutes, not seconds** (API calls, provisioning time)
- Failures are **harder to detect** (a misconfigured security group doesn't crash — it silently allows unauthorized access)

You need a testing strategy calibrated for infrastructure: fast checks for syntax, moderate checks for logic, and slow but thorough checks for real deployments.

### The Terraform Testing Pyramid

```
                    ╱╲
                   ╱  ╲
                  ╱ E2E╲         Slowest, most expensive
                 ╱(Terra-╲       Real AWS resources created
                ╱  test)  ╲      Run: weekly or per release
               ╱──────────╲
              ╱ Integration╲     Medium speed, moderate cost
             ╱  (terraform  ╲    terraform plan on real backend
            ╱    test / plan)╲   Run: per PR
           ╱──────────────────╲
          ╱   Static Analysis   ╲    Fast, free
         ╱  (validate, fmt,     ╲   No API calls
        ╱   tflint, checkov)     ╲  Run: on every commit
       ╱──────────────────────────╲
```

## 2. Core Concept — Deep Dive

### Level 1: Static Analysis (Fastest, Free)

**`terraform validate`** — syntax and type checking
```bash
$ terraform validate
Success! The configuration is valid.

# Catches:
# - Missing required arguments
# - Wrong attribute names
# - Type mismatches (string where number expected)
# - Invalid references
# Does NOT catch:
# - Invalid CIDR ranges, wrong AMI IDs (requires API)
```

**`terraform fmt`** — formatting consistency
```bash
$ terraform fmt -check -diff
# Returns non-zero exit code if files need formatting
# In CI: fail the build if code isn't formatted

$ terraform fmt -recursive
# Format all .tf files in subdirectories too
```

**tflint** — Terraform-specific linting
```bash
# Install
brew install tflint   # macOS
# OR
choco install tflint  # Windows

# Run
tflint --init
tflint
```

```hcl
# .tflint.hcl
plugin "aws" {
  enabled = true
  version = "0.31.0"
  source  = "github.com/terraform-linters/tflint-ruleset-aws"
}

rule "terraform_naming_convention" {
  enabled = true
  format  = "snake_case"
}

rule "terraform_documented_variables" {
  enabled = true
}

rule "terraform_documented_outputs" {
  enabled = true
}
```

**tflint catches:**
- Invalid instance types (`t3.micr0` — typo)
- Deprecated resource arguments
- Missing descriptions on variables/outputs
- Naming convention violations

**checkov / tfsec** — security scanning
```bash
# Install
pip install checkov

# Scan
checkov -d .

# Output:
# Check: CKV_AWS_79: "Ensure Instance Metadata Service Version 1 is not enabled"
# PASSED for: aws_instance.web
#
# Check: CKV_AWS_88: "Ensure EC2 instance has no public IP"
# FAILED for: aws_instance.web
```

**Security checks include:**
- S3 buckets without encryption
- Security groups open to 0.0.0.0/0
- RDS without encryption at rest
- EC2 without IMDSv2
- Missing logging configurations

### Level 2: Plan-Based Testing (Medium Speed)

**`terraform plan` as a test** — verify expected changes
```bash
# Plan and save output
terraform plan -out=tfplan -input=false

# Convert to JSON for programmatic analysis
terraform show -json tfplan > plan.json

# Check: plan should create exactly 5 resources
cat plan.json | jq '[.resource_changes[] | select(.change.actions[0] == "create")] | length'
# Expected: 5

# Check: no resources are being destroyed
cat plan.json | jq '[.resource_changes[] | select(.change.actions[] == "delete")] | length'
# Expected: 0
```

**`terraform test` (native, Terraform 1.6+)** — HCL-based tests
```hcl
# tests/vpc_test.tftest.hcl

variables {
  vpc_cidr     = "10.99.0.0/16"
  environment  = "test"
  project_name = "test-project"
}

run "create_vpc" {
  command = plan   # or apply (creates real resources)

  assert {
    condition     = aws_vpc.main.cidr_block == "10.99.0.0/16"
    error_message = "VPC CIDR does not match expected value"
  }

  assert {
    condition     = aws_vpc.main.enable_dns_support == true
    error_message = "DNS support should be enabled"
  }

  assert {
    condition     = length(aws_subnet.public) == 2
    error_message = "Expected 2 public subnets"
  }
}

run "validate_tags" {
  command = plan

  assert {
    condition     = aws_vpc.main.tags["Environment"] == "test"
    error_message = "Environment tag should be 'test'"
  }

  assert {
    condition     = aws_vpc.main.tags["ManagedBy"] == "terraform"
    error_message = "ManagedBy tag should be 'terraform'"
  }
}
```

```bash
# Run native tests
terraform test

# Output:
# tests/vpc_test.tftest.hcl... in progress
#   run "create_vpc"... pass
#   run "validate_tags"... pass
# Success! 2 passed, 0 failed.
```

**Test file location:**
```
project/
├── main.tf
├── variables.tf
├── outputs.tf
└── tests/
    ├── vpc_test.tftest.hcl
    ├── compute_test.tftest.hcl
    └── security_test.tftest.hcl
```

### Level 3: End-to-End Testing (Slowest, Most Thorough)

**Terratest (Go-based)** — creates real infrastructure, validates, destroys
```go
// test/vpc_test.go
package test

import (
    "testing"
    "github.com/gruntwork-io/terratest/modules/terraform"
    "github.com/stretchr/testify/assert"
)

func TestVpcModule(t *testing.T) {
    t.Parallel()

    terraformOptions := terraform.WithDefaultRetryableErrors(t, &terraform.Options{
        TerraformDir: "../modules/vpc",
        Vars: map[string]interface{}{
            "vpc_cidr":     "10.99.0.0/16",
            "environment":  "test",
            "project_name": "terratest",
        },
    })

    // Destroy at end of test
    defer terraform.Destroy(t, terraformOptions)

    // Create real infrastructure
    terraform.InitAndApply(t, terraformOptions)

    // Validate outputs
    vpcId := terraform.Output(t, terraformOptions, "vpc_id")
    assert.NotEmpty(t, vpcId)
    assert.Contains(t, vpcId, "vpc-")

    publicSubnets := terraform.OutputList(t, terraformOptions, "public_subnet_id_list")
    assert.Equal(t, 2, len(publicSubnets))
}
```

```bash
# Run Terratest
cd test
go test -v -timeout 30m
```

**Cost and time considerations:**
- Each test run creates real AWS resources → costs money
- VPC creation: ~30 seconds
- RDS creation: ~10 minutes
- Full stack: 15-30 minutes
- Run in a dedicated test AWS account with spending limits

## 3. CI/CD Integration

### Complete Testing Pipeline

```yaml
# .github/workflows/terraform-test.yml
name: Terraform Tests

on:
  pull_request:
    branches: [main]

jobs:
  # ─── Level 1: Static (every PR, < 1 min) ───
  static-analysis:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: hashicorp/setup-terraform@v3
      
      - name: Format Check
        run: terraform fmt -check -recursive
      
      - name: Validate
        run: |
          terraform init -backend=false
          terraform validate
      
      - name: TFLint
        uses: terraform-linters/setup-tflint@v4
        with:
          tflint_version: latest
      - run: |
          tflint --init
          tflint
      
      - name: Security Scan
        uses: bridgecrewio/checkov-action@master
        with:
          directory: .

  # ─── Level 2: Plan Test (every PR, ~2 min) ───
  plan-test:
    needs: static-analysis
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: hashicorp/setup-terraform@v3
      
      - name: Configure AWS
        uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: ${{ secrets.TEST_ROLE_ARN }}
          aws-region: us-east-1
      
      - name: Terraform Test
        run: terraform test

  # ─── Level 3: E2E (weekly or release, ~30 min) ───
  e2e-test:
    if: github.event.label.name == 'run-e2e' || github.event_name == 'schedule'
    needs: plan-test
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-go@v5
        with:
          go-version: '1.22'
      
      - name: Configure AWS
        uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: ${{ secrets.E2E_TEST_ROLE_ARN }}
          aws-region: us-east-1
      
      - name: Run Terratest
        working-directory: test
        run: go test -v -timeout 30m
```

## 10. Interview Preparation

### Q1: *"How do you test Terraform code?"* (Senior)

**Strong Answer:**
"I use a testing pyramid. Base layer: static analysis on every commit — `terraform validate`, `terraform fmt -check`, `tflint` for best practices, and `checkov` for security. Middle layer: `terraform test` with plan-mode assertions on every PR — validates resource counts, tag values, and CIDR calculations without creating real resources. Top layer: Terratest for critical modules — creates real infrastructure in a test account, validates outputs and connectivity, then destroys. E2E tests run weekly or on releases due to cost and time."

### Q2: *"Your module works in dev but fails in prod. How do you prevent this?"* (Senior)

**Strong Answer:**
"The root cause is usually different variables between environments. Prevention: `terraform test` with production-like variable values catches this at the plan level. Stronger: Terratest that applies the module with prod-like config in a test account. Strongest: staging environment that mirrors prod — apply there first and validate before prod. Also, variable validation catches invalid values early without any infrastructure creation."

### Q3: *"What's the difference between `terraform validate` and `terraform test`?"* (Mid)

**Strong Answer:**
"`validate` checks syntax and type correctness — it's fast and free, doesn't need credentials, and catches obvious errors. `terraform test` runs actual plan or apply operations with custom assertions — it can verify that a VPC gets the right CIDR, that subnets are created in the right AZs, and that outputs have expected values. `test` is more powerful but slower and may need AWS credentials."

## 13. Cheat Sheet

```bash
# Level 1: Static (fast, free)
terraform fmt -check -recursive
terraform validate
tflint --init && tflint
checkov -d .

# Level 2: Plan tests (medium, cheap)
terraform test                          # Native (1.6+)
terraform plan -json | jq '...'        # Custom plan analysis

# Level 3: E2E (slow, costs money)
cd test && go test -v -timeout 30m     # Terratest
```

```hcl
# Native test file: tests/example.tftest.hcl
run "test_name" {
  command = plan    # or apply
  
  variables {
    key = "value"
  }
  
  assert {
    condition     = expression
    error_message = "failure description"
  }
}
```
